# Source: News / injury RSS feeds (news)

- **Reachable:** partial. Terms and robots pages were reachable for the major publishers surveyed; only the GDELT DOC API RSS-format endpoint was fetched as a data sample because it is explicitly positioned as a free/open programmatic data source.
- **Access method:** public RSS/news survey; one public GDELT DOC API RSS-format probe.
- **Auth required:** none for the survey or GDELT probe. Guardian Open Platform requires an API key for API use; not used in this task.
- **requires_secret:** false for the selected Phase 3 metadata-only option.
- **License / terms URL:** BBC terms/content/robots: `https://www.bbc.co.uk/usingthebbc/terms-of-use/`, `https://www.bbc.co.uk/usingthebbc/terms/can-i-use-bbc-content/`, `https://www.bbc.co.uk/robots.txt`; Guardian feeds/terms/Open Platform: `https://www.theguardian.com/help/feeds`, `https://www.theguardian.com/help/terms-of-service`, `https://open-platform.theguardian.com/access/`; ESPN/Disney terms: `https://disneytermsofuse.com/english/`; Sky/Sky Sports terms/robots: `https://www.sky.com/help/articles/skycom-terms-and-conditions`, `https://www.skysports.com/robots.txt`; GDELT: `https://www.gdeltproject.org/`, `https://www.gdeltproject.org/data.html`, `https://blog.gdeltproject.org/gdelt-doc-2-0-api-debuts/`; official bodies checked for caution: `https://inside.fifa.com/terms-of-service`, `https://inside.fifa.com/robots.txt`, `https://www.uefa.com/termsconditions/`, `https://www.uefa.com/robots.txt`.
- **Allowed use (1 line):** Store metadata/aggregates only - headline/title, source link, timestamp, source, and derived counts/flags; never store or redistribute full article text, and do not use publisher RSS feeds for automated collection unless the publisher terms expressly allow the specific use.
- **Endpoint(s) / URL(s) probed:**
  - GDELT DOC API RSS-format query: `https://api.gdeltproject.org/api/v2/doc/doc?query=%22soccer+injury%22&mode=artlist&maxrecords=3&timespan=1day&sort=datedesc&format=rssarchive`
  - Terms/robots metadata pages listed above.
  - Publisher RSS feed URLs were surveyed but not fetched: BBC Sport football `https://feeds.bbci.co.uk/sport/football/rss.xml`; Guardian football `https://www.theguardian.com/football/rss`; ESPN soccer candidate `https://www.espn.com/espn/rss/soccer/news`; Sky Sports football candidate `https://www.skysports.com/rss/12040`.
- **Schema (key columns/fields):** GDELT RSS sample observed `title`, `link`, and `pubDate`; the committed excerpt also reserves optional `guid` when present. Candidate publisher RSS feeds should be treated as headline/summary metadata only if separately licensed; do not persist `description`, article body, images, or extracted text.
- **Row / record count in sample:** GDELT RSS probe returned 1 item for the narrow one-day test query. No publisher RSS items were downloaded.
- **Date range / freshness (latest record date):** GDELT sampled item `pubDate` was `21 Jun 2026 02:00:00 +0000`; GDELT states its main streams update every 15 minutes. BBC/Guardian/ESPN/Sky update cadence is continuous/newsroom-driven but no project-safe SLA was selected in this survey.
- **Frozen?** no. News/RSS sources are live, but they are intentionally deferred and should not be a core model dependency.
- **2026 World Cup relevance:** medium. News can supply optional qualitative injury/squad/lineup context before and during the tournament, but it should not feed the probability path until Phase 3 compliance, entity resolution, and leakage controls exist.
- **Gotchas:** A HTTP 200 is not valid RSS until content type and XML shape confirm it; the GDELT sample validated `application/rss+xml` plus `<rss>`. GDELT initially returned HTTP 429 for a broader query, so any future collector needs a strict rate limit and narrow queries. Publisher terms are restrictive: BBC robots/terms warn against scraping/systematic extraction and require permission for AI/computer analysis; Guardian RSS is personal non-commercial and Guardian terms restrict automated collection, data aggregation/mining, ML/AI uses, and commercial use without approval; Disney/ESPN terms prohibit automated extraction/data-set building; Sky Sports did not expose a clear RSS reuse grant in this survey; FIFA/UEFA official platforms are private/non-commercial and restrict automated/systematic collection. Headlines are noisy and can be multilingual; headline hits are not official injury status.
- **Recommended phase:** 3
- **Retention recommendation:** raw_retention_days=0 for publisher RSS feeds unless a source-specific license is approved. For GDELT Phase 3 experiments, raw_retention_days=7 for gitignored development samples only; bronze_retention_days=365 for normalized metadata/aggregate tables containing `observed_at`, `source`, `title`, `url`, `published_at`, entity links, and derived counts/flags. Never redistribute full article text.
- **Sample saved at:** GDELT raw RSS sample `discovery/samples/news/gdelt-doc-rss-sample.xml` (gitignored); committed metadata-only excerpt `discovery/findings/d9-news-gdelt-excerpt.json`.
- **Status:** usable with caveats for Phase 3 metadata-only monitoring; publisher RSS feeds are not selected for automated ingestion without permission.

## Phase 3 recommendation

Use GDELT, if anything, as the first Phase 3 news signal because it is designed as a programmatic news-data layer and can return RSS/XML metadata. Keep the signal outside the core probability path: it can power optional qualitative context such as "recent injury/squad headlines mentioning Team X" or aggregate counts by team/source/time window.

Do not store or redistribute full article text. Do not scrape publisher article pages. Do not train on publisher body text. Do not treat headlines as confirmed injury status without later human/source validation.
